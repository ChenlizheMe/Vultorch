    15 — Space Battle¶

    Example file: examples/15_space_battle.py

    All our previous examples used tensors as visualizations of some computation. But a tensor is just a 2-D grid of numbers — there's nothing stopping you from using it as a frame buffer for a tiny pixel game.

    This chapter builds a complete space shooter where every pixel is drawn by writing into a torch.Tensor on the GPU. You'll learn real-time keyboard input, pixel-perfect canvas rendering, and how to record gameplay to video.

    What we're building¶

    A retro shoot-'em-up at 160×200 resolution:

    Area Content
    Left Controls — score, lives, difficulty slider, record button
    Right Game — pixel-art space battle, nearest-neighbour upscaled

    Controls: W/A/S/D to move, Space to shoot.

    New friends¶

    New thing What it does Why it matters
    ui.is_key_down(key) Test if a keyboard key is currently held Real-time game input without polling events
    ImGui key codes Constants like KEY_W = 568 imgui.h enum values starting at 512
    filter="nearest" Nearest-neighbour upscaling on canvas Pixel-perfect rendering — no blur
    canvas.save(path) Save current frame as an image One-click screenshots
    Tensor as framebuffer Write RGB values directly into slices Full creative freedom — sprites, particles, text

    Keyboard input¶

    Vultorch exposes ImGui's key state through the ui module:

    from vultorch import ui

KEY_W = 568  # imgui.h: ImGuiKey_W
KEY_A = 546
KEY_S = 564
KEY_D = 549
KEY_SPACE = 525

if ui.is_key_down(KEY_A):
    player_x -= speed
if ui.is_key_down(KEY_D):
    player_x += speed

    The key codes come from the ImGuiKey enum in imgui.h, where named keys start at 512. The most useful ones for games:

    Key Code Key Code
    W 568 Space 525
    A 546 Escape 527
    S 564 Enter 528
    D 549 Tab 512

    Pixel art sprites¶

    We define sprites as ASCII art, then parse them into coordinate lists:

    PLAYER_SPRITE = [
    "    X    ",
    "   XXX   ",
    "  XXXXX  ",
    " XX X XX ",
    " XXXXXXX ",
    "XXX   XXX",
    "X       X",
]

def parse_sprite(lines):
    pts = []
    for r, line in enumerate(lines):
        for c, ch in enumerate(line):
            if ch == "X":
                pts.append((r, c))
    return pts

    To draw a sprite, stamp its points into the tensor:

    for dr, dc in PLAYER_PTS:
    r, c = player_y + dr, player_x + dc
    if 0 <= r < H and 0 <= c < W:
        fb[r, c, 0] = 0.2   # R
        fb[r, c, 1] = 0.8   # G
        fb[r, c, 2] = 1.0   # B

    Nearest-neighbour rendering¶

    A 160×200 game would look blurry if upscaled with bilinear filtering. Use filter="nearest" to get crisp pixels:

    canvas = panel.canvas("game", filter="nearest")

    Recording gameplay¶

    From the previous chapter, adding recording is one line:

    ctrl.record_button(canvas, "gameplay.gif", fps=30)

    Play for a while, then stop recording — you get a pixel-perfect GIF of your gameplay session.

    Game architecture¶

    The Game class manages all state:

    • Player: position, speed
    • Bullets: list of [x, y], move upward 4 px/frame
    • Enemies: list of [x, y, hp], spawn from top, move down
    • Explosions: list of [x, y, timer], expand then fade
    • Stars: background parallax scrolling

    Each frame: game.update(move_x, move_y, shoot) then game.render(display[:, :, :3]). The tensor is the frame buffer.

    Full code¶

    import random
import torch
import vultorch
from vultorch import ui

if not torch.cuda.is_available():
    raise RuntimeError("This example needs CUDA")

device = "cuda"

GW, GH = 160, 200
PLAYER_W, PLAYER_H = 9, 7
BULLET_W, BULLET_H = 1, 4
ENEMY_W, ENEMY_H = 7, 5

KEY_W = 568
KEY_A = 546
KEY_S = 564
KEY_D = 549
KEY_SPACE = 525

C_BG       = (0.05, 0.05, 0.12)
C_PLAYER   = (0.2, 0.8, 1.0)
C_BULLET   = (1.0, 1.0, 0.3)
C_ENEMY    = (1.0, 0.3, 0.2)
C_ENEMY2   = (1.0, 0.6, 0.1)
C_STAR     = (0.6, 0.6, 0.7)
C_EXPLODE  = (1.0, 0.9, 0.4)

PLAYER_SPRITE = [
    "    X    ",
    "   XXX   ",
    "  XXXXX  ",
    " XX X XX ",
    " XXXXXXX ",
    "XXX   XXX",
    "X       X",
]

ENEMY_SPRITE = [
    " XXXXX ",
    "XXXXXXX",
    "XX X XX",
    "XXXXXXX",
    " X   X ",
]


def parse_sprite(lines):
    pts = []
    for r, line in enumerate(lines):
        for c, ch in enumerate(line):
            if ch == "X":
                pts.append((r, c))
    return pts


PLAYER_PTS = parse_sprite(PLAYER_SPRITE)
ENEMY_PTS = parse_sprite(ENEMY_SPRITE)


class Game:
    def __init__(self):
        self.px, self.py = GW // 2, GH - 20
        self.speed = 2
        self.bullets, self.enemies, self.explosions = [], [], []
        self.stars = [(random.randint(0, GW-1), random.randint(0, GH-1),
                       random.uniform(0.3, 1.0)) for _ in range(60)]
        self.score, self.lives = 0, 3
        self.game_over = False
        self.spawn_timer, self.spawn_rate = 0, 30
        self.bullet_cooldown, self.frame = 0, 0
        self.difficulty = 1.0

    def reset(self):
        self.__init__()

    def update(self, move_x, move_y, shoot):
        if self.game_over:
            return
        self.frame += 1
        self.px = max(PLAYER_W//2, min(GW-1-PLAYER_W//2, self.px + move_x*self.speed))
        self.py = max(PLAYER_H, min(GH-5, self.py + move_y*self.speed))
        if shoot and self.bullet_cooldown <= 0:
            self.bullets.append([self.px, self.py - PLAYER_H//2 - 1])
            self.bullet_cooldown = 6
        self.bullet_cooldown = max(0, self.bullet_cooldown - 1)

        self.bullets = [b for b in self.bullets if (b.__setitem__(1, b[1]-4) or b[1] > 0)]

        self.spawn_timer -= 1
        if self.spawn_timer <= 0:
            ex = random.randint(ENEMY_W//2, GW-1-ENEMY_W//2)
            self.enemies.append([ex, -ENEMY_H, 1])
            self.spawn_timer = max(8, int(self.spawn_rate / self.difficulty))

        # ... (collision detection, enemy movement, explosions, stars)

    def render(self, fb):
        fb[:,:,0] = C_BG[0]; fb[:,:,1] = C_BG[1]; fb[:,:,2] = C_BG[2]
        # draw stars, enemies, bullets, explosions, player ...
        if self.game_over:
            fb *= 0.3


game = Game()
view = vultorch.View("15 - Space Battle", 900, 750)
ctrl = view.panel("Controls", side="left", width=0.30)
game_panel = view.panel("Game")

display = vultorch.create_tensor(GH, GW, channels=4, device=device,
                                 name="game", window=view.window)
display[:, :, 3] = 1.0
canvas = game_panel.canvas("game", filter="nearest")
canvas.bind(display)


@view.on_frame
def update():
    move_x = (1 if ui.is_key_down(KEY_D) else 0) - (1 if ui.is_key_down(KEY_A) else 0)
    move_y = (1 if ui.is_key_down(KEY_S) else 0) - (1 if ui.is_key_down(KEY_W) else 0)
    shoot = ui.is_key_down(KEY_SPACE)
    game.update(move_x, move_y, shoot)
    with torch.no_grad():
        game.render(display[:, :, :3])


@ctrl.on_frame
def draw_ctrl():
    global game
    ctrl.text(f"FPS: {view.fps:.0f}")
    ctrl.separator()
    ctrl.text(f"Score: {game.score}")
    hearts = "♥ " * game.lives + "♡ " * (3 - game.lives)
    ctrl.text(f"Lives: {hearts}")
    if game.game_over:
        ctrl.text_colored(1, 0.3, 0.3, 1, "GAME OVER!")
        if ctrl.button("Restart", width=150):
            game = Game()
    ctrl.separator()
    game.difficulty = ctrl.slider("Difficulty", 0.5, 3.0, default=1.0)
    game.speed = ctrl.slider_int("Player Speed", 1, 5, default=2)
    ctrl.separator()
    ctrl.record_button(canvas, "gameplay.gif", fps=30)
    if canvas.is_recording:
        ctrl.text_colored(1, 0.3, 0.3, 1, "● REC")
    ctrl.separator()
    if ctrl.button("Screenshot", width=150):
        canvas.save("screenshot.png")


view.run()

    Try it¶

    python examples/15_space_battle.py

    Use WSAD to dodge enemies, Space to fire. Crank up the difficulty slider if it's too easy. Hit Record to capture your best run.
